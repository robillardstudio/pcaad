Adding files in the p5 editor:

  • create an empty file with Sketch/Add File
  • show/hide files with the grey “>” button over the line numbers
  • find and select the file in “Sketch Files” on the left after created
  • edit the file just like you would the js file

Note you can also put files in a subfolder with Sketch/Add subfolder:

  • examples often use a folder called assets
  • then instead of loading “numbers.txt” say “assets/numbers.txt”
  • a common mistake is writing “assets/file.txt” when you don’t actually have a folder called “assets” but you saw that’s what the example did so…

Local servers

Well explained on the p5.js site: Local server

This is NOT an issue if you are using the p5.js editor or other web-server-based code editor (e.g. codepen, glitch, github, etc).

This IS an issue when you download your code and try to run it in your browser on your machine. The easiest solution is the Web Server for Chrome Extension.

If you are curious and want to know more:

  • same-origin policy
  • cross-site scripting
  • SimpleHTTPServer
