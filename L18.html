Images as arrays of pixels

An Image is less of a programming concept and more of a data structure that is particularly useful in our art/design domain.

We’ve learned about colors, arrays, and Objects. Combining them, we get p5’s approach to creating, accessing, and filtering Image objects.

The image class has properties that you might expect an image to have, e.g., a width, height, and a bunch of pixels that each has a color.

It also has methods (like functions) that let you check and change individual pixels, mask, filter, blend, the image, etc.

Changing Images: filters and pixels

You can tint, filter, and blend Images.

Note in these examples the order of tint() and filter() before or after image()! Tint acts the way we’re used to (before you draw) while filter gets applied to the already-drawn canvas.

Note I think some people have found some of these processes to be very slow (i.e. not something you can do 60 times/second). If you think this might be true, see how it behaves at a lower resolution (smaller canvas).

Here are examples showing transparency and alpha masks.

The color values inside images at specific pixels can be accessed. You can get the value of a pixel based on its (X,Y) location. This pointillism example uses the get() function to set the circle colors.

Advanced: Image Generation

To access the values of individual pixels (as an array of colors) you must first call the loadPixels function.
