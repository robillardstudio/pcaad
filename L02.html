Tools for Writing Code:

We’re going start with a simple web site that can run your code, the p5.js editor.

Other more general web interfaces some people use might include:

  • codepen - developers
  • github - open source software
  • glitch - from learning to web apps

You might someday prefer to create your code on your own machine instead of only relying on a remote server. This is how most programmers work.

The p5.js site discusses using VS Code as an alternative code editor.

I still use github’s atom but it was discontinued at the end of 2022.

Professional programmers use full IDEs like Visual Studio and XCode.

You can technically use ANY text editor, but they won’t know you are writing code, so formatting it correctly and finding problems will be a struggle and it will be less fun.

If you don’t have any reason not to, you should be fine sticking with the p5.js editor.

Setting and using variables

  • we can store numbers: var numHouses = 10;
  • we can store words: let name = “bob”;
  • using “var” vs “let” has to do with something called “scope”. For the moment use either.
  • there are different types of data: ints, floats, strings, booleans
  • some programming languages are “untyped”, “weakly typed”, or “dynamically typed”
  • javascript doesn’t care. Can say “let x = 5” and then “x =”bob”;” right after

In p5.js you’ll learn about a bunch of predefined (built-in) variables.

Some examples include “height” and “mouseX” which get set by p5.js, not you. e.g. “height” always contains the height in pixels of your drawing canvas, while mouseX always tells you the horizontal position of the mouse cursor.

Take a look at Conditionals and Interactivity

We’ve mentioned variables and conditionals.
One place these intersect is with more complex “booleans”.

These are TRUE or FALSE statements.
What if we wanted to say, “if we’re in THIS region, then do something?

This might look like:

if((x > 50) && (x < 100) && (y > 60) && (y < 90) ) { /* do stuff */ }

// if X is between 50 and 100, AND y is between 60 and 90, THEN…

&& means AND
|| means OR
! means NOT
