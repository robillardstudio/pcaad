A bit about shape transforms

A thing everyone always asks about eventually: using scale, rotate, and translate. This is not really programming knowledge, it’s really computer graphics stuff. But here’s a bit of help.

There are two things you need to consider:

  • scale and rotate use the coordinate system’s ORIGIN: (0,0)
    • shapes grow (and shrink) from the upper left corner
    • shapes rotate around the upper left corner
  • the order you scale, rotate, translate does different things

See rotate reference

It might help to think of a process (“algorithm”) like this:

  • create a shape at the origin
  • scale it to be the size you want
  • rotate the shape so it is oriented how you want
  • now move (translate) it where you want it.

Make sense? You’ve probably worked this way in other software.

BUT to program this, you’ll write the code backwards! (sorry)

  translate(width / 2, height / 2);  // move to center
  rotate(PI / 3.0);                  // rotate 60 degrees
  scale(4, 1);                       // make it wider
  rect(-5, -5, 10, 10);              // draw a square centered at (0,0)

See the bottom of this Transform example

Also, older transform example

Think of this the same way as when we use the fill() or stroke() color functions. We are changing the state of the system BEFORE we draw stuff.

The code above could be read like a poorly told story:

  • “WHATEVER we eventually draw, move it to the center!”
  • Ok, here’s what we’re going to draw…
  • It’s something that should be rotated 60 degrees (“PI/3” radians)…
  • Ok, here it is…
  • Oh and also scale its width by 4…
  • It’s a rectangle that was drawn in the upper left corner!

You could also think of this as like translate( rotate( scale( rect() ) ) ) but you can’t actually write that.

You do NOT have to do this for this (or any) assignment. But you may find it useful for moving your shapes around, or modifying copies of them.

Some of you might also be interested to read about the push() and pop() functions.
