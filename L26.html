The HTML DOM (Document Object Model)

Today we’re going to take a look at how the p5.js skills you’ve been learning can interact with the rest of a web page, since we’ve been working in a web browser.

This isn’t really a general programming concept, more of an application. But it’s a really obvious one, given the programming environment we’ve been learning. So think of it as more applied programming practice that you might find useful.

So first we’ll glance at how web pages/browsers work, and then how you can connect p5.js to a few types of web page elements.

First, maybe take a quick glance at What is HTML? if you’re not already feeling confident about that. You’ve got tags around your text. And these are frequently in a hierarchy (inside a document, you’ve got a paragraph that maybe contains a list that maybe has one word in bold.)

And note there is also HTML for things like buttons and images. There also might be attributes controlling the style of things.

Now give this a quick skim and just get the general idea: What is HTML DOM

Hopefully the DOM kind of looks like OOP with objects that have their own properties and methods: a button has a size property and can respond to a click.

That article also shows you some vanilla Javascript for accessing different elements in the DOM:

var myElement = document.getElementById("intro");

In most cases p5.js will have functions to make this easier for us, but know it’s a thing. As always, please feel free to post questions of the form, “what the heck…?” in today’s discussion thread (or email me, or set up a zoom appointment…)
